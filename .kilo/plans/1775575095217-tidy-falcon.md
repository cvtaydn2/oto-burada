# Oto Burada - Sistem Analiz Raporu

## Yönetici Özeti

Oto Burada, mobil öncelikli, güven odaklı ve sadece araba ilanlarına odaklanan ücretsiz bir ilan pazaryeri MVP'sidir. Proje, Next.js App Router mimarisi üzerinde inşa edilmiş olup Supabase backend platformunu kullanmaktadır. MVP fazı ve Persistence Upgrade tamamlanmış, mevcut aşamada UI Modernization çalışmaları devam etmektedir.

---

## 1. Proje Kimliği

### Temel Değerler
- **Güven**: Doğrulanmış satıcılar, moderasyon sistemi
- **Sadelik**: Gereksiz karmaşıktan uzak, odaklı deneyim
- **Hız**: Mobilde hızlı filtreleme, 2 dakikada ilan oluşturma
- **Şeffaflık**: Açık fiyatlar, detaylı araç bilgileri

### Başarı Kriterleri
1. Kullanıcı 2 dakikadan kısa sürede ilan oluşturabilmeli
2. Kullanıcı 3 etkileşimden azda ilana ulaşabilmeli
3. Kod tabanı yeni geliştirici tarafından anlaşılabilir olmalı
4. Uygulama build, lint ve typecheck'ten geçmeli

---

## 2. Teknoloji Yığını

### Frontend
- **Framework**: Next.js 16.2.2 (App Router)
- **Dil**: TypeScript (strict mode)
- **Stil**: Tailwind CSS 4
- **UI Kütüphanesi**: shadcn/ui (Radix UI tabanlı)
- **Form**: React Hook Form + Zod
- **Veri Yönetimi**: TanStack Query v5
- **İkonlar**: lucide-react

### Backend
- **Auth**: Supabase Auth
- **Veritabanı**: Supabase Postgres
- **Depolama**: Supabase Storage
- **Güvenlik**: Row Level Security (RLS)

### Geliştirme Araçları
- ESLint, TypeScript compiler
- Playwright (test)
- PostgreSQL client scripts

## 4. Veritabanı Şeması

### Tablolar
- **profiles**: Kullanıcı profilleri (full_name, phone, city, role)
- **listings**: İlanlar (title, brand, model, year, mileage, fuel, transmission, price, city, status)
- **listing_images**: İlan görselleri (storage_path, public_url, sort_order)
- **favorites**: Kullanıcı favorileri
- **reports**: Raporlar (reason, status)
- **admin_actions**: Moderasyon audit trail

### Enum Tipleri
- user_role: user, admin
- listing_status: draft, pending, approved, rejected, archived
- fuel_type: benzin, dizel, lpg, hibrit, elektrik
- transmission: manuel, otomatik, yari_otomatik
- report_reason: fake_listing, wrong_info, spam, other
- report_status: open, reviewing, resolved, dismissed
- moderation_action: approve, reject, review, resolve, dismiss, archive

### RLS Politikaları
- profiller: Kullanıcı kendi profilini veya admin okuyabilir/güncelleyebilir
- listings: Onaylı ilanlar herkese açık, sahip kendi ilanlarını yönetir
- favorites: Kullanıcı yalnızca kendi favorilerini yönetir
- reports: Kullanıcı yalnızca kendi raporlarını oluşturur, admin güncelleyebilir
- admin_actions: Yalnızca admin erişebilir

## 6. API Rotaları

### Listings
- `POST /api/listings` - İlan oluştur
- `GET /api/listings` - İlanları listele
- `GET /api/listings/[id]` - İlan detay
- `DELETE /api/listings/[id]` - İlan arşivle
- `POST /api/listings/images` - Görsel yükle
- `DELETE /api/listings/images` - Görsel sil

### Favorites
- `GET /api/favorites` - Favorileri oku
- `POST /api/favorites` - Favori ekle/kaldır

### Reports
- `POST /api/reports` - Rapor oluştur

### Admin
- `POST /api/admin/listings/[id]/moderate` - Moderasyon işlemi
- `PATCH /api/admin/reports/[id]` - Rapor durumu güncelle

### Migrations
- `POST /api/migrations/legacy-sync` - Legacy cookie backfill
